# Contract Note

## Форма endpoint

В задании имитируется legacy-контроллер с двумя публичными точками входа:

- `CommercialProductController::update(mixed $id, array $data): JsonResponse`
- `CommercialProductController::patch(mixed $id, array $data): JsonResponse`

## Допустимые форматы payload

Оба метода принимают один из следующих вариантов:

```php
['entity' => ['name' => 'Светильник']]
```

или

```php
['name' => 'Светильник']
```

## Известные поля домена

Сейчас значимыми считаются поля:

- `name`
- `count`
- `price`
- `type`
- `comment`
- `image_uuid`

## Текущий контракт успешного ответа

При успешном обновлении контроллер возвращает:

- HTTP-статус `200`
- JSON payload с обновлённым продуктом

Сейчас payload продукта включает:

- `id`
- `name`
- `count`
- `price`
- `type`
- `comment`
- `image_uuid`
- `image`

## Текущая legacy-особенность

Если `count < 1`, текущая legacy-реализация удаляет продукт вместо обычного обновления.

Это странное поведение, но оно намеренно оставлено в задании, потому что вдохновлено реальным кодом. Его можно сохранить, а можно заменить на более адекватный вариант, если решение объяснено и покрыто тестами.
